# 全局规则

1、在编写任何代码前，请先描述你的方案并等待批准，如果需求不明确，在编写任何代码之前必须提出澄清问题

2、如果一项任务需要修改超过3个文件，请先停下来，将其分解成更小的任务

3、编写代码后，列出可能出现的问题，并建议相应的测试用例来覆盖问题

4、档发现bug时，首先要编写一个能够修复bug的测试，然后不断修复它，直到测试通过为止

5、每次纠正你之后，就在CLAUDE.md文件中添加一条新规则，这样就不会发生这种情况了

6、每次修改代码或着写代码请根据当前项目的功能、需求、代码结构等补充或者修改项目级CLAUDE.md

7、单文件不超过200行代码 如果超过必须拆分

8、禁止硬编码，端口、url、配置 → 环境变量

9、新建前必须搜索有没有类似功能，有就扩展，不要重复造轮子

10、禁止静默异常，所有错误必须记录

11、禁止向后兼容，旧代码直接删除

12、尽可能要有详细的中文注释，至少每个函数都得有注释

13、你需要不断更新项目里的CLAUDE.md文件 让它适配当前项目的开发

14、.claude目录不要上传到github
